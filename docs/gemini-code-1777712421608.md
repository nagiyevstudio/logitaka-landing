# Промпт для ИИ-разработчика: Компонент ModelAdvisorWidget

**Роль и задача:**
Ты — Senior Frontend разработчик. Твоя задача — создать интерактивный UI-компонент `ModelAdvisorWidget`, который помогает пользователям выбрать правильные ИИ-модели на основе их профессии.

### 1. Состояния компонента (State)
Компонент должен хранить два состояния:
* `selectedCategory` (по умолчанию `null` или первая категория).
* `selectedRole` (по умолчанию `null` или первая роль в выбранной категории).

### 2. Логика интерфейса (UI Layout)
Используй минималистичный, чистый дизайн (Tailwind CSS предпочтителен).
* **Шапка:** Заголовок «Подбор моделей под ваши задачи». Подзаголовок: «Выбери, чем ты обычно занимаешься, и мы покажем, какие модели закроют 100% твоих задач без переплат».
* **Уровень 1 (Выбор сферы):** Набор горизонтальных табов или селектор с названиями категорий (`categoryLabel`).
* **Уровень 2 (Выбор роли):** Появляется ряд кнопок (Chips/Pills) с названиями ролей (`roleLabel`). Активная должна подсвечиваться.
* **Уровень 3 (Результаты):** Появляется после выбора роли с плавной анимацией (fade-in).
    * **Блок "Обоснование":** Текстовый абзац `rationale` (выдели его курсивом или легкой рамкой).
    * **Блок "По типу задач":** Сетка из 3 карточек (Ежедневные, Важные, Сложные). В карточке: Название задачи, Название модели (крупно), и индикатор стоимости.
    * **Блок "Внутри Logitaka":** Сетка из 2 или 3 карточек (Совещание, Дискуссия, DevChat). Если `devchat: null`, эту карточку рендерить не нужно. В карточке: Название контура, Рекомендуемая модель, индикатор стоимости, короткий текст `desc`.

### 3. Цветовая логика для индикатора стоимости
Значки доллара (`cost`) должны раскрашиваться строго по этой схеме:
* `$` — Светло-серый или базовый синий (Минимальный расход)
* `$$` — Зеленый (Экономный расход)
* `$$$` — Желтый/Оранжевый (Умеренный расход)
* `$$$$` — Светло-красный (Высокий расход)
* `$$$$$` — Темно-красный (Экстремальный расход)
Создай helper-функцию `getCostColor(cost)`.
