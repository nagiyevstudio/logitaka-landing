# Landing Assets Guide

This static landing works without images, but it has dedicated visual slots where real assets can later be inserted.

Recommended files:

1. `hero-visual.webp`
Purpose:
- first-screen emotional visual
- should support the message `chaos -> structure`

Suggested concept:
- one human figure or silhouette in relation to an abstract system
- layered panels, lines, notes, signals, or interface fragments
- controlled intensity, not chaotic collage

Suggested format:
- `1600x1200` or larger
- dark-friendly and light-friendly composition

2. `context-visual.webp`
Purpose:
- support the `live context` section

Suggested concept:
- conversation fragments turning into structured nodes
- projects, plans, decisions, links, memory
- should feel like continuity and retained context

Suggested format:
- `1600x1000` or larger

3. `dev-team-visual.webp`
Purpose:
- support the developer section

Suggested concept:
- code/editor/terminal atmosphere mixed with project memory
- architecture notes, decisions, task flow, instruction handoff
- should feel operational, not gamer/neon cliché

Suggested format:
- `1600x1000` or larger

Optional:

4. `founder-portrait.webp`
Purpose:
- founder note section

Suggested format:
- `1200x1600` or larger
- serious portrait, neutral or dark background

Notes:
- current HTML uses abstract placeholders instead of broken image tags;
- once assets are ready, the placeholder blocks in `index.html` can be replaced with real `<img>` tags or CSS background images.
